## Dynamic, const e final

## Dynamic

O `dynamic` é para quando você não sabe (ou não quer dizer) qual o tipo da variável. 

Ele desativa a verificação estática de tipos do Dart. Ele tbm pode mudar de tipo durante a execução.

- **Uso:** Em respostas de APIs JSON onde a estrutura é incerta ou em funções genéricas complexas
- → Muito comum ao ler objetos JSON utilizando `Map<String, dynamic>`, onde a chave do JSON é uma `String` e o valor pode ser de qualquer tipo (String, int, List, outro Map, etc).
    
    → Isso acontece porque um JSON pode conter estruturas mistas e aninhadas, tornando impossível definir um único tipo fixo para os valores.
    

Exemplo de JSON com estrutura variada:

```json
{"name":"Vitalflux.com",
"domain":["Data Science","Mobile","Web"],
"noOfArticles":[{"type":"data science","count":50},{"type":"web","count":75}]}
```

Nesse caso, ao converter para Dart, algo como `Map<String, dynamic>` é usado porque:

- `"name"` → String
- `"domain"` → List<String>
- `"noOfArticles"` → List<Map<String, dynamic>>
- **Problema:** Você perde a segurança do código, pois o Dart desativa a verificação. Se tentar chamar um método que não existe, o erro só aparece quando o app rodar (Runtime Error).
    - **"NoSuchMethodError":**
    Se você definir uma variável como dynamic e tentar chamar o método `.voar()`, o Dart não vai te avisar que o erro existe até que o usuário clique no botão e o aplicativo feche sozinho (crash).
    - Quando é chamado uma função ou método inválido. Aparece quando uma função dynamic ou um método dynamic resulta em um tipo de argumento inválido ou uma lista. Para non-dynamic, os tipos static impede esses tipos de argumentos inválidos (https://api.flutter.dev/flutter/dart-core/NoSuchMethodError-class.html).
    

```dart
dynamic valor = "TipoString";
valor = 20; // Tipo Int associado na variável.
```

### Exemplos

```dart
// Quando uma função pode receber diferentes tipos de entrada e processá-los

void imprimirDado(dynamic dado) {
  print("O valor é: $dado");
}

void main() {
  imprimirDado("String"); // O valor é: String
  imprimirDado(123); // O valor é: 123  
}
```

```dart
// Quando você precisa de uma lista que aceita diferentes tipos de dados

void main() {
  List<dynamic> listaMista = ["Texto", 10, true, 3.14];
  
  listaMista.forEach((item) {
    print("Item: $item - Tipo: ${item.runtimeType}");
  });
  // Item: Texto - Tipo: String
  // Item: 10 - Tipo: int
  // Item: true - Tipo: bool
  // Item: 3.14 - Tipo: double
}
```

## Final

O `final` é usado para valores que não mudam depois de definidos, mas que você só descobre quais são quando o código está rodando. Só pode ser definida uma vez, independente se o novo valor for do mesmo tipo. Ela é definida no **tempo de execução** (runtime).

- **Exemplo:** O horário atual ou o resultado de uma requisição ao banco de dados.
- **Diferença com o 'var':**
Se você usar `final nome = 'João';`, o Dart ainda pratica a **inferência de tipo**. Ele sabe que é uma String, mas garante que você nunca mais atribua outro valor a `nome`.

```dart
final String nomeUsuario = obterNomeDoBanco(); 
nomeUsuario = "Novo Nome"; // Mesmo sendo string linha dá erro pq não pode ser alterado.
```

### Exemplos

```dart
// Comum em 'StatelessWidgets' para passar dados ao construtor

class SaudacaoWidget extends StatelessWidget {
  final String nome;

  SaudacaoWidget({required this.nome});

  @override
  Widget build(BuildContext context) {
    return Text("Olá, $nome");
  }
}

// 'StatelessWidgets': A widget that does not require mutable state.
// A stateless widget is a widget that describes part of the user
// interface by building a constellation of other widgets that describe
// the user interface more concretely.
```

Referência: https://api.flutter.dev/flutter/widgets/StatelessWidget-class.html

```dart
// Usado para garantir que os dados não mudem após o objeto ser criado 
class Produto {
  final String nome;
  final double preco;

  Produto(this.nome, this.preco);
}

void main() {
  final prod = Produto("Teclado", 150.0);
  print(prod.nome);
  
  // Se fizer: 
  prod = Produto("Mouse", 50.0); // vai dar erro!
}
```

## Const

O `const` também está relacionado a imutabilidade. Mas diferente do `final` você já sabe o valor dela antes mesmo do app abrir e também sabe que ele será imutável. É uma constante em **tempo de compilação**.

- **Performance:** O Flutter ama `const` porque ele economiza memória.
    
    → Como o Flutter depende fortemente de uma árvore de widgets para representar a UI, quando marcamos um widget como `const`, o Flutter reconhece que aquele objeto é imutável.
    
    → Isso permite que o Flutter reutilize a mesma instância do objeto ao invés de criar uma nova a cada rebuild, tornando a performance mais fluida.
    
    → O `const` permite pré-alocação em tempo de compilação, o que otimiza a criação do objeto e reduz o trabalho em tempo de execução (runtime).
    
    → Como consequência, há menor geração de lixo (garbage) na memória durante o runtime.
    
    → Além disso, reduz o risco de comportamento inesperado e facilita a depuração (debug), pois usar `const` comunica explicitamente a intenção de criar um objeto imutável. Isso torna o código mais fácil de entender, já que o objeto não irá mudar após ser criado.
    
    → Um dos princípios centrais do Flutter é minimizar o número de rebuilds desnecessários para atualizar a UI. Widgets marcados como `const` ajudam diretamente nesse princípio.
    
    → A filosofia do Flutter se apoia fortemente em conceitos de programação funcional, onde a imutabilidade é um aspecto chave.
    
- **Obrigatoriedade Recursiva:**
Para um objeto ser `const`, tudo dentro dele também precisa ser `const`. Você não pode ter uma lista `const` que contém uma variável que pode mudar.

- **Curiosidade: Canonicalização de Objetos**
**(Isso aqui talvez seja interessante de demonstrar em sala, mostraria entendimento da linguagem como ele pediu, mas precisa ser melhor visto também)**
    - Esta é a parte mais técnica: quando você cria duas variáveis `const` com o mesmo valor, o Dart aponta ambas para o **exatíssimo mesmo lugar na memória**.
    
    > Exemplo: Se você tiver dez mil botões com `const EdgeInsets.all(8.0)`, todos eles estarão usando a mesma única instância na memória RAM. Isso economiza bateria e processamento do celular.
    > 

### Exemplos

```dart
// Sem const: O Flutter pode 
// reconstruir este widget 
// desnecessariamente

Container(
  padding: EdgeInsets.all(8.0),
  child: Text('Olá Mundo'),
)

// Com const: O Flutter cria uma
// única instância e a reutiliza

const Padding(
  padding: EdgeInsets.all(8.0),
  child: Text('Olá Mundo'),
)
```

---

### Tempo de Alocação (Compilação vs. Execução)

A diferença fundamental entre **const** e **final** é o momento em que o valor é definido. O **const** precisa ser conhecido enquanto você está escrevendo o código (ou quando o computador está compilando ele). O **final** aceita valores que só aparecem quando o usuário interage com o app.

### Flexibilidade de Tipagem

Enquanto o **dynamic** permite que uma variável comece como um número e termine como um texto, o **final** e o **const** travam o tipo assim que o valor é atribuído. Eles são focados em segurança, enquanto o dynamic é focado em flexibilidade.

### Impacto na Performance do Flutter (Verificar)

O Flutter possui um mecanismo chamado **"Repaint Boundary"**. 

Quando você usa `const` nos seus componentes visuais (widgets), você está dizendo para o Flutter: "Não precisa redesenhar este pedaço da tela, ele nunca vai mudar". Isso torna a rolagem de listas (scroll) muito mais fluida.

### Parâmetros obrigatorios
```dart
  apresentar(String nome, int idade) {
    return "meu nome é $nome e eu tenho $idade anos."; <<< interpolação : utilizando o símbolo $ para variáveis simples ($nome) ou ${} para expressões complexas (${1+1})
}

void main () {
  print(apresentar(21, "tayna")); <<< ERRO! Pois a ordem dos valores importam para o parametro da função
}

>>>> se não declarar o tipo da variável no parâmetro como:
apresentar( nome,  idade) {
    return "meu nome é $nome e eu tenho $idade anos."; <<< interpolação : utilizando o símbolo $ para variáveis simples ($nome) ou ${} para expressões complexas (${1+1})
}
>>>>> vai funcionar, porém pode ficar estranho o retorno como: "meu nome é 21 e eu tenho tayna anos."
```

### Parâmetros opcionais
```dart
  apresentar(String nome, [int? idade = 0]) { << torna o parametro opcional, porem tem que declarar como null ou um valor default para caso n seja passado nenhum valor
    print("meu nome é $nome");
  if(idade != null) {
    print("tenho $idade anos");
  }
}

void main () {
 apresentar("tayna");

  apresentar("tayna", 20);
}
```
